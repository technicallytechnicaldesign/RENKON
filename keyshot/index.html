KeyShot Scripts

Automation for the Creo → KeyShot pipeline — batch views, turntables, and camera-driven reveal animations, written against the lux / luxmath scripting API. Headless-compliant where noted, so batch stages can run unattended on a licensed KeyShot install.

Inventory

2ABAT
2a_BAT_STD_VIEW_AA01.py
Studios-first multi-view batch render. Imports every model of a chosen extension from a folder, applies an optional material template, then renders every Studio (camera + environment pairing) defined per part — falls back to raw cameras only when a part has no Studios. Logs a CSV manifest.

2a_BAT_TURNTABLE_AA01.py
Studios-first 360° turntable batch. Same Studio-aware camera/lighting selection as the std-view script; spins each imported part and encodes a small mp4 per part. Kept as a continuous spin on purpose — the "fade between angles" idea is planned as a separate 3_ post-process script instead of a change here.
Current
2BANI
2b_ANI_HERO_REVEAL_AA01.py
Automated hero-reveal shot: single monotonic zoom (in or out) into a camera/Studio's own framing, with optional vertical crane, HDRI rotation, brightness ramp, and synced Model Set animation playback. Orbit/arc was deliberately removed (Rodrigues' formula degenerates when the rotation axis aligns with the camera-to-object direction); ground-setter method name and getAnimationInfo()'s return shape are unconfirmed on your build — the script probes and reports which worked. See in-file docstring for full design notes.

Conventions

  • Filenames: {PREFIX}_{AREA}_{NAME}_{REV}.py — prefix is pipeline stage (0_ pre-check · 1_ pre-render/helper · 2a_ batch render · 2b_ animation · 3_ post-process · 4_ post-check), area is a 3-letter code (BAT, ANI, HLP…), REV steps a letter for a breaking change and a number for a routine one.
  • KeyShot scripting is Python 3.10+, using the lux / luxmath modules.
  • Headless CLI execution is supported for anything marked HEADLESS COMPLIANT — good for unattended batch renders.
  • Headless automation still needs a machine with a licensed KeyShot install; GitHub-hosted Actions runners can't run it.
  • Studios (camera + environment + image style, paired) are preferred over raw Cameras throughout — raw cameras leave lighting as whatever's already active, which is wrong for parts with multiple lighting setups.
  • Binary scene files (.bip, .ksp…) are tracked via Git LFS — name versions clearly (chair_v3.bip) rather than relying on diffs.
